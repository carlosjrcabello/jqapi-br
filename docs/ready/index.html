.ready()

Categorias: Events > Document Loading

.ready( handler ) Retorna: jQuery

Descrição: Especifica a função a ser executada quando o DOM estiver totalmente carregado.

  • versão em que foi adicionada: 1.0.ready( handler )

    handlerFunção a ser executada depois que o DOM estiver pronto.

Enquanto o Javascript fornece o evento load para executar um código quando a página é renderizada, este evento não é disparado enquando todos ativos como imagens sejam totalmente recebidos. Na maioria dos casos, o script pode ser executado assim que toda a hierarquia DOM for construída. O handler passado para .ready() é garantido que será executado após o DOM estiver pronto, por isso este é geralmente o melhor lugar para anexar todos os outros manipuladores de evento e executar outros códigos jQuery. Quando se utiliza scripts que dependem do valor propriedades de stilo CSS, é importante fazer referência a folhas de estilo externas ou estilo de embutir elementos antes de referenciar os scripts.

Nos casos quando o código depende de ativos carregados (como por exemplo, se as dimensões de uma imagem são necessárias), o código deve ser colocado em um manipulador (handler) em vez de um evento load.

O método .ready() é geralmente incompatível com o atributo <body onload="">. Se o load precisar ser usado, então não use os métodos .ready() ou jQuery .load() para atachar manipuladores de evento load para a janela ou mais itens específicos, como imagens.

Todos as três sintaxes seguintes são equivalentes:

  • $(document).ready(handler)
  • $().ready(handler) (esta não é recomendada)
  • $(handler)

Há também o $(document).bind("ready", handler). Este se comporta de maneira semelhanete ao método .ready() mas com uma exceção: Se o evento .ready() tenha sido disparado e você tente usar .bind("ready") o manipulador vinculado não será executado. Manipuladores para .ready() ligados desta forma são executados depois de qualquer obrigado pelos outros três métodos acima.

The método .ready() somente pode ser chamado por um objeto jQuery combinando o documento corrente, de modo que o seletor pode ser omitido.

O método .ready() é tipicamente usado com uma função anônima: 

$(document).ready(function() {
  // Manipulador para .ready() chamado.
});

Que é equivalente a chamada:

$(function() {
 // Manipulador para .ready() chamado.
});

Se .ready() é chamado depois que o DOM tenha sido inicializado, o novo manipulador passado será executado imediatamente.

Alias o nome de domínio (namespace) do jQuery

Quando usamos outra biblioteca Javascript, nos podemos chamar $.noConflict() para evitar problemas com nome de domínio (namespace). Quando a função é chamada, o atalho $ não fica mais disponível, nos forçando a escrever jQuery cada vez que normalmente escreveríamos $. No entando o manipulador passado para o método .ready() pode pegar um argumento, que é passado o objeto jQuery global. Isto significa que podemos mudar o nome do objeto dentro do contexto de nosso manipulador .ready() sem afetar outros códigos. 

jQuery(document).ready(function($) {
  // Code using $ as usual goes here.
});

Example:

Display a message when the DOM is loaded.

<!DOCTYPE html>
<html>
<head>
  <style>p { color:red; }</style>
  <script src="http://code.jquery.com/jquery-git.js"></script>
  <script>
  $(document).ready(function () {
  $("p").text("The DOM is now loaded and can be manipulated.");
});
  </script>

</head>
<body>
  <p>Not loaded yet.</p>

</body>
</html>
