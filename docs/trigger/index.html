.trigger()

Categorias: Events > Event Handler Attachment

.trigger( eventType, extraParameters ) Retorna: jQuery

Descrição: Executa todos os manipuladores e comportamentos ligados aos elementos combinados para o tipo de evento dado.

  • versão em que foi adicionada: 1.0.trigger( eventType, extraParameters )

    eventType: Uma String contento o tipo do evento Javascript, como click ou submit.

    extraParameters: Um array de parâmetros adicionais para passar para o manipulador de eventos.

  • versão em que foi adicionada: 1.3.trigger( event )

    event: Um objeto jQuery.Event.

Alguns gerenciadores de eventos anexados com .bind() ou um dos métodos de atalho são disparados quadno o evento correspondente ocorre. Eles podem ser acionados manualmente, no entanto, com o método .trigger(). A chamada de .trigger() executa os gerenciadores na mesma ordem em que seriam se o evento foram provocadas naturalmente pelo usuário: 

$('#foo').bind('click', function() {
      alert($(this).text());
    });
    $('#foo').trigger('click');

A partir do jQuery 1.3, eventos .trigger()ados na bolha da árvore DOM; um manipulador de eventos pode parar a borbulhar, retornando falso do manipulador ou chamando o método .stopPropagation() sobre o objeto de evento passado para o evento. Contudo .trigger() simula a ativação de um evento, completa com um objeto de evento sintetizados, não replica perfeitamente um evento que ocorre naturalmente.

Para disparar manipuladores via jQuery sem também disparar o evento nativo, use .triggerHandler().

Quando definimos um evento customizado usando o método .bind(), o segundo argumento passado para o .trigger() pode tornar-se útil. Por exemplo, suponha que temos ligado um manipulador para o evento 'custom' para o nosso elemento ao invés de constuí-lo com o evento click como fizemos acima: 

$('#foo').bind('custom', function(event, param1, param2) {
  alert(param1 + "\n" + param2);
});
$('#foo').trigger('custom', ['Custom', 'Event']);

o objeto de evento é sempre passado como o primeiro parâmetro para um manipulador de evento, mas se parâmetros adicionais forem especificados durante .trigger() chama como eles estão aqui, esses parâmetros vão ser repassados ​​para o manipulador também.

Observe a diferença entre os parâmetros extras que estamos passando aqui e o parâmetro eventData para o método .bind(). Ambos são mecanismos para passar informações a um manipulador de eventos, mas o argumento extraParameters passado para .trigger() permite que a informação seja determinada no momento em que o evento é acionado, enquanto o argumento eventData do .bind() exige a informação já esteja calculada no momento em que o manipulador é ligado.

Exemplos:

Example: Clique no botão #2 também aciona o click do botão #1. 

<!DOCTYPE html>
<html>
<head>
  <style>

button { margin:10px; }
div { color:blue; font-weight:bold; }
span { color:red; }
</style>
  <script src="http://code.jquery.com/jquery-git.js"></script>
</head>
<body>
  <button>Button #1</button>
<button>Button #2</button>
<div><span>0</span> button #1 clicks.</div>

<div><span>0</span> button #2 clicks.</div>
<script>
$("button:first").click(function () {
update($("span:first"));
});
$("button:last").click(function () {
$("button:first").trigger('click');

update($("span:last"));
});

function update(j) {
var n = parseInt(j.text(), 10);
j.text(n + 1);
}
</script>

</body>
</html>

Demonstração:

Exemplo: Submitar o primeiro formulário sem usar a função .submit(), tente 

$("form:first").trigger("submit")

Exemplo: Submitar o primeiro formulário sem usar a função .submit(), tente: 

var event = jQuery.Event("submit");
$("form:first").trigger(event);
if ( event.isDefaultPrevented() ) {
// Execute uma ação..
}

Exemplo: Para passar dados arbitrários para um evento: 

$("p").click( function (event, a, b) {
// quando um click normal é disparado, a e b estão indefinidos.
// para um gatilho como abaixo se refere a um "foo" e b refere-se a "bar"

} ).trigger("click", ["foo", "bar"]);

Exemplo: Para passar dados arbitrários para um evento: 

var event = jQuery.Event("logged");
event.user = "foo";
event.pass = "bar";
$("body").trigger(event);

Exemplo: Forma alternativa para transmitir dados através de um objeto de evento: 

$("body").trigger({
type:"logged",
user:"foo",
pass:"bar"

});
